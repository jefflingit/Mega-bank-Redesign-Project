Artifact models

Artifact models for 4 participants

首頁 / 頂端服務標籤

首頁標籤難以理解，使用者找尋困難

本組觀察到兆豐首頁標籤用詞對於使用者理解難度高，使用者不了解「線上結匯」即為「線上換匯」，也不清楚「外幣現鈔」也能進入「線上換匯」

所有使用者都遇到同樣的狀況，會在首頁因找不到直觀的按鈕而停留許久，甚至有受訪者跳出網頁到Google直接搜尋「兆豐銀行 線上換匯」。

另外，兆豐銀行網站首頁標籤階層太多，各階層包含的服務也相當豐富，但卻缺乏清楚指引。

因此大部分受訪者表示不清楚「線上換匯」應該屬於哪個標籤之下，也造成在首頁不知所措的狀況。

首頁 / 外匯幣率資訊

標籤描述不清，造成操作不直觀

其中一位受訪者有瀏覽到首頁下方的匯率資訊，然而該資訊欄中包含的「線上結匯」超連結，該受訪者不會主動點擊。

目前歸納出兩大原因，首先是該「線上結匯」標籤缺乏超連結的直觀操作指引。

第二則是受訪者並不瞭解「線上結匯」為「線上換匯」，因該受訪者點擊該資訊欄中的「買賣外幣」，卻不是他所需要的服務。

外幣現鈔頁面

「兌換幣別」內容不完整，受訪者心中產生疑惑。

本次訪談邀請每位受訪者操作兆豐銀行線上換匯的流程，所設定的情境為兌換500,000韓元。

但在外幣現鈔頁面「兌換幣別」內容中，剛好缺少韓圓的幣別，儘管下方備註欄有提及相關資訊，依舊有一位受訪者對於該頁面的操作產生疑惑。

總歸來說因為幣別內容不完整、備註訊息不明顯，才會導致操作上不夠直觀。

線上結匯頁面

大部分受訪者對於該頁面的操作是抱持著「不確定性」。

有一位受訪者認為「立即申購」按鈕背景的白色資訊，並非是重要且必要的資訊。

有其他兩位受訪者對於「立即申購」按鈕較難理解，一位受訪者不曉得該按紐為「線上換匯」服務按鈕，因此點擊前猶豫一段時間。

另一位受訪者則認為應提供後續的流程介紹，才能幫助他確定該服務是他需要的以及他適不適合線上換匯。

線上結匯服務說明與約定事項

使用者容易忽略約定事項內重要內容及下方勾選按鈕。

此約定事項頁面主要從受訪者得到兩項痛點。第一是合約內容呈現方式，因過多文字並且許多重要資訊隱藏在內容下方，造成一位受訪者沒有耐心閱讀並理解這些合約內容，因此錯過許多重要訊息。

第二個痛點為勾選按鈕不明顯，造成一位受訪者忽略按鈕，導致操作流程不順。

第一步-輸入申購資料 / 領取地點及日期

分行選項資訊不足，造成使用者難以選擇。

進行選取領鈔分行的操作時，目前網頁只會顯示「分行名稱」，較難選擇離宿舍最近的分行，因此讓使用者跳出該網頁到Google尋找分行位置。而受訪者在隨意選取一個分行後，才發現原來網頁會顯示在下方。

而受訪者在隨意選取一個分行後，才發現原來地址與Google maps會顯示在下方，但對於受訪者來說覺得不夠直觀與方便，佔用太多時間。

第一步-輸入申購資料 / 購買人資料

購買人資料格式提示不明確，導致受訪者操作錯誤。

本組訪問到共計三位受訪者填寫「出生年月日」時，因不清楚身邊的「(yyyy/MM/dd)」提醒代表需填入的格式，因此輸入時產生不確定感，甚至受訪者重複出錯。

第一步-輸入申購資料 / 結匯幣別與金額

表格元素表達不完整，受訪者無法直觀操作。

「結匯幣別與金額」表格中呈現的元素，大部分受訪者認為不夠完整並且缺乏解釋，因此造成操作上的困難。

受訪者主要提到問題有以下幾點：

1.數字後缺乏單位表示

2.下拉式選單數字、單位缺乏解釋

3.不理解「現鈔CASH」標籤

第一步-輸入申購資料 / 注意事項

注意事項頁面內容過多，有受訪者選擇略過而忽略重要資訊。

與「線上結匯服務說明與約定事項」頁面情形相同，受訪者表示該頁面內容太多，會選擇忽略不看。

第一步-輸入申購資料 / 外匯申報資料

標籤文字過於專業、缺乏解釋，使一部分受訪者難以理解。

「申報目的地國別」欄位從受訪者身上觀察到兩大問題。第一是有受訪者表示並不理解此資料欄位意義，是否為欲旅遊國。第二是有受訪者表示選單過多國家，但卻沒有輔助性搜尋方法，讓該受訪者於選取國別時花費時間較長。

第一步-輸入申購資料 / 付款方式-信用卡

信用卡銀行選擇不足

一位受訪者使用「信用卡」作為付款方式時，認為目前提供的信用卡選擇過少，操作不順。

第一步-輸入申購資料 / 付款方式-銀行轉帳

注意事項容易被使用者跳過

一位受訪者表示付款方式下的「注意事項」欄位容易被忽略，在操作後才發現原來這些注意事項是會因為不同付款方式而變動，因此認為是相對重要的資訊。

第三步-交易結果確認

重要資訊字體過小

在「交易結果確認」頁面上，有一位受訪者表示重要資訊字體過小，如「需要攜帶本人身分證或護照」。

第三步-交易結果確認 / 台灣pay繳費QR Code

台灣Pay付款QR Code清晰易懂

一位選用台灣Pay作為付款方式的受訪者表示，該頁面顯示出足夠大、清楚的解釋對他來說容易理解該如何進行付款。

第三步-交易結果確認 / 繳款與領鈔注意事項

注意事項內容過多，使用者會選擇挑過。

一位受訪者表示，「繳款與領鈔注意事項」頁面中內容過多，讓該受訪者會選擇忽略，因此錯失繳款注意事項、領款方式。

第三步-交易結果確認 / 領取資訊與加購服務

受訪者對於加購產品感到疑惑

一位受訪者表示該頁面下的加購產品「購賣旅平險、我要兌換外語課程」等表示疑惑。

而另一位受訪者提到自己會有存取訂單編號和繳費證明的習慣，但在此頁面會不小心忽略領取資訊。
